  • ElementHeaderFooter(_:isEquivalent:element:background:pressedBackground:configure:)

    Provides a way to create a HeaderFooter for your Blueprint elements without requiring the creation of a new BlueprintHeaderFooterContent struct.

    Most arguments on this method are not required – you must only provide an input and an element provider.

    Note

    This initializer is helpful if you have to nest an existing element that needs to be used in a single place, without needing to define an entirely new type.

    If your header or footer is to be used in more than one place, it is recommended that you create a BlueprintHeaderFooterContent struct to share logic to avoid duplicate code.

  • ElementItem(_:id:isEquivalent:element:background:selectedBackground:configure:)

    Provides a way to create an Item for your Blueprint elements without requiring the creation of a new BlueprintItemContent struct.

    Most arguments on this method are not required – you must only provide an input, a key path for the backing identifier value, and an element provider.

    Note

    This initializer is helpful if you have to nest an existing element that needs to be used in a single place, without needing to define an entirely new type.

    If your item is to be used in more than one place, it is recommended that you create a BlueprintItemContent struct to share logic to avoid duplicate code.

    Example

    ElementItem(podcast, id: \.name) { lhs, rhs in
    lhs.value != rhs.value
} element: { podcast, info in
    PodcastElement(podcast: podcast)
} background: { podcast, info in
    Box(...)
} selectedBackground: { podcast, info in
    Box(...)
} configure: { item in
    item.selectionStyle = .tappable
}
