ListScrollPositionInfo

public struct ListScrollPositionInfo : Equatable

Information about the current scroll position of a list, including which edges of the list are visible, and which items are visible.

This is useful within callback APIs where you as a developer may want to perform different behavior based on the position of the list, eg, do you want to allow an auto-scroll action, etc.

  • visibleContentEdges(includingSafeAreaEdges:)

    Used to retrieve the visible content edges for the list’s content.

    Eg, for vertical lists:

    • If the list is scrolled all the way to the bottom, the visible edges are the left, right, and bottom.
    • If the list is scrolled all the way to the top, the visible edges are the left, right, and top.

    You can use this method to determine how and where the list is scrolled if you want to modify behavior based on the current scroll position.

    Safe Area Insets

    You can control whether safeAreaInsets should be taken into account via the includingSafeAreaEdges parameter.

    Generally, you want to include the safeAreaInsets for the top, left, and right, but may want to exclude the bottom if you consider the bottom edge visible if it’s visible below the home indicator on a home button-less iPhone or iPad.
